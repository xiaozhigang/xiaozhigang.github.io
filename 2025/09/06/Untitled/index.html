MetaGPT 重写学习路线图

flowchart TD
    A[Step-1: 环境准备] --> B[Step-2: 最小 Agent]
    B --> C[Step-3: 多角色协作]
    C --> D[Step-4: 任务分解与执行流]
    D --> E[Step-5: 工具调用与记忆模块]
    E --> F[Step-6: 项目级应用场景]

    subgraph Step-1: 环境准备
        A1[熟悉 Python 基础]
        A2[了解 Asyncio / 并发]
        A3[安装依赖 & 熟悉代码结构]
    end

    subgraph Step-2: 最小 Agent
        B1[实现简单输入 → 输出]
        B2[理解 Prompt 模板化]
    end

    subgraph Step-3: 多角色协作
        C1[角色设定: PM / Eng / QA]
        C2[Agent 之间消息传递]
    end

    subgraph Step-4: 任务分解与执行流
        D1[学习 Workflow / Pipeline]
        D2[实现任务分解器]
        D3[调度多个 Agent 协同]
    end

    subgraph Step 5: 工具调用与记忆模块
        E1[调用外部 API / 搜索]
        E2[集成数据库 / 知识库]
        E3[实现短期 & 长期记忆]
    end

    subgraph Step-6: 项目级应用场景
        F1[写一个需求 → 产出代码 Demo]
        F2[扩展到多任务并行]
        F3[优化日志 & 可视化监控]
    end

mindmap
root((MetaGPT 重写学习路线))
0. 目标与范围
定义目标: 复现一个最小的多智能体协作框架
基准任务: “做一个计算器” → PRD → 代码 → 运行
产出: 跑通端到端最小案例
1. 环境 & 基础设施
语言: Python 3.10+
依赖: asyncio, httpx/requests, pydantic, faiss/duckdb(可选)
目录: src/ tests/ examples/ plugins/
配置: .env settings.py 日志
2. 最小 Agent MVP
Agent类: name, role, prompt
LLM抽象: LLMClient.call(prompt, tools?)
动作: act(input) → output
Demo: 回答/总结/改写
验收: CLI 示例 + 单测
3. 角色系统 (Role)
Role基类: receive() → think() → act() → produce()
消息模型: Message{sender, role, content, type}
具体角色: PM, Engineer, QA(可先2个)
验收: PM 产出 PRD → Eng 产出代码
4. 调度器 Team
Team: agents[], run(goal)
流程: 串行/并行/循环(重试/评审)
通信: 事件总线 pub/sub 或黑板(共享状态)
验收: 目标→PRD→代码 的端到端输出
5. SOP / Workflow
Step节点: 输入/提示/输出 Schema
编排: Pipeline/Graph(有向图)
可重放: trace/日志/快照
验收: 同一 SOP 可复用在不同任务
6. 记忆层 Memory
短期: 会话窗口/对话历史裁剪
长期: 向量库(FAISS) + 文档分块
RAG: Retriever + Chunker + Ranker
验收: 能回忆历史与检索知识
7. 工具 & 执行器
Tool接口: name, schema, run()
常用工具: web_search, code_exec, db_query
执行沙盒: 超时/资源限额/docker(可选)
安全: 白名单, 速率限制, 预算
8. 多智能体协作
协议: 任务卡/工单, 投票/仲裁
拓扑: PM→ENG→QA→Reviewer
同步: 黑板/消息队列
验收: 多 Agent 协作完成复杂任务
9. 观测 & 评测
日志: 结构化日志 + TraceID
指标: tokens/latency/cost/success rate
可视化: trace graph/调用链
评测: 基准任务通过率/回归集
10. 项目化落地
接口: CLI/HTTP(gRPC/REST)
配置: 插件注册/角色模板/工具清单
持久化: sqlite/redis(队列/缓存)
部署: docker compose
11. 扩展方向
计划器: 反思/自纠错/计划-执行-反思
编排: 兼容 LangGraph/CrewAI
缓存: prompt/检索/结果缓存
成本控制: 预算/配额/降级
12. 里程碑(示例)
Week1: 最小Agent + CLI
Week2: 角色系统 + Team
Week3: SOP + Memory
Week4: 工具 + 协作
Week5: 观测 + 项目化
